== Security Module Guide

=== Authorization Model
* Primary model: RBAC with first-class matrix permissions (`resource:action`).
* Effective authority flow: `User -> Role -> Privilege`.
* URL ACL flow: `Url(endpoint, method) -> Privilege`.

=== Core Permission Convention
* Use lower-case action codes, for example:
** `user:read`, `user:create`, `user:update`, `user:delete`
** `role:read`, `role:create`, `role:update`, `role:delete`
** `privilege:read`, `privilege:assign`
** `url:read`, `module:manage`, `matrix:read`

=== Runtime Enforcement Layers
* HTTP security chain authenticates bearer JWT and enforces stateless access.
* `ACLFilter` validates request URI + method against DB-mapped privileges and is the primary runtime authorization engine for `/api/**` routes.
* `@PreAuthorize` is reserved for hard-stop governance endpoints, not routine CRUD authorization.

=== Seeded Baseline (Development)
* Privileges are seeded by `PrivilegeLoader`.
* Roles are seeded by `RoleLoader`.
* URL-to-privilege mappings are seeded by `UrlsLoader` and synced by `EndpointSync`.
* Users are seeded by `UserLoader`:
** `superadmin` (SUPER_ADMIN)
** `admin` (ADMIN)
** `user` (USER)

=== Matrix Capability API
* Endpoint: `GET /api/me/permissions`
* Returns:
** current user identity
** assigned roles
** flattened authorities
** matrix grouped by resource and action

=== Policy Drift Tooling
* Startup checker: `PolicyDriftStartupCheck`
* Runtime report API: `GET /api/policy/drift`
* Drift detector validates:
** `@PreAuthorize(hasAuthority(...))` permission format
** controller authority vs URL policy mapping consistency
** undeclared permission code warnings for policy convention drift
* Config flags in `application.yml`:
** `security.policy.drift.enabled`
** `security.policy.drift.fail-on-drift`

=== Admin Console
* Path: `/assets/admin/index.html`
* Uses matrix authorities from `/api/me/permissions` to show/hide actions.
* Superadmin/admin get management screens according to granted permissions.

=== Extending Security
1. Add permission constant in `PermissionCodes`.
2. Add seed in `PrivilegeLoader`.
3. Attach to role(s) in `RoleLoader`.
4. Map endpoint+method in `UrlsLoader`.
5. Use `@PreAuthorize` only if the endpoint is a governance hard-stop.
6. Use `permissions.can("resource:action")` in admin UI.
